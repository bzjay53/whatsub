OAuth 문제 해결

OAuth 인증 오류가 발생할 경우:

  1. Google Cloud Console에 로그인하세요.
  2. 사용자 인증 정보 만들기 > OAuth 클라이언트 ID를 선택하세요.
  3. 클라이언트 ID가 이미 있는 경우 클릭하여 확인하세요.
  4. 애플리케이션 유형이 Chrome 확장 프로그램으로 설정되었는지 확인하세요.
  5. 클라이언트 ID가 1060430808730-ic272t33ti0dje3ig6q3ge5l8895fq6n.apps.googleusercontent.com와 일치하는지 확인하세요.
  6. 승인된 리디렉션 URI에 반드시 chrome-extension://[확장프로그램ID]/oauth2 형식의 URI가 추가되어 있어야 합니다.
  7. 개발자 콘솔 로그에서 [Whatsub] OAuth 리디렉션 URI (oauth2 접미사 포함): 항목을 확인하고 이 값을 Google Cloud Console에 정확히 등록하세요.

리디렉션 URI 불일치 (redirect_uri_mismatch) 오류:

이 오류는 Google Cloud Console에 등록된 리디렉션 URI와 실제 확장 프로그램의 리디렉션 URI가 일치하지 않을 때 발생합니다. 크롬 개발자 도구(F12)의 콘솔 탭에서 "[Whatsub] OAuth 리디렉션 URI (oauth2 접미사 포함):" 로그를 확인하고 이 URI를 Google Cloud Console의 OAuth 클라이언트 ID 설정에 추가하세요.

자주 묻는 질문

Q: 자막이 표시되지 않아요.

A: 동영상이 재생 중인지 확인하고, 자막 토글 버튼이 활성화되어 있는지 확인하세요.

Q: 자막 언어를 변경하고 싶어요.

A: 설정 탭에서 원하는 언어로 변경할 수 있습니다.

Q: 자막이 정확하지 않아요.

A: 노이즈 감소 기능을 활성화하고, 가능한 음성이 명확한 환경에서 사용해보세요.
